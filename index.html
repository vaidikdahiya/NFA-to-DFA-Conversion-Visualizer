01. Finite Automata

A Finite Automaton (FA) is a mathematical model of computation used to recognize patterns in strings. It is defined by a 5-tuple:

M = (Q, Σ, δ, q₀, F) Q — Finite set of states Σ — Input alphabet (finite set of symbols) δ — Transition function q₀ — Start state (q₀ ∈ Q) F — Set of accept / final states (F ⊆ Q)

02. NFA vs DFA

NFA — Nondeterministic

Can have multiple transitions for the same symbol, and ε-transitions that consume no input. May be in multiple states simultaneously.

DFA — Deterministic

Exactly one transition per symbol per state. No ε transitions. Always in exactly one state. Easier to implement but may have exponentially more states.

Both NFA and DFA recognize the same class of languages — Regular Languages. Every NFA has an equivalent DFA.

03. Subset Construction Algorithm

The Powerset (Subset Construction) algorithm converts an NFA to an equivalent DFA. Each DFA state represents a set of NFA states.

Algorithm: 1. Compute ε-closure({q₀}) → initial DFA state 2. For each unprocessed DFA state S and each symbol a ∈ Σ: move(S, a) = { t | ∃s ∈ S, (s, a, t) ∈ δ } ε-closure(T) = T ∪ { states reachable via ε from T } New DFA state = ε-closure(move(S, a)) 3. A DFA state is accepting if it contains any NFA accept state 4. Repeat until no new DFA states are discovered

Worst case: an NFA with n states may produce a DFA with up to 2ⁿ states.

04. DFA Minimization (Hopcroft)

Hopcroft's algorithm minimizes a DFA by partition refinement: states that behave identically are merged.

Hopcroft's Algorithm: 1. Start with partition P = {F, Q \ F} (accept vs non-accept) 2. For each group G in P, for each symbol a ∈ Σ: splitter(G, a) = split G where all states go to same partition 3. Replace G with its subgroups. Repeat until stable. 4. Each final partition = one state in the minimized DFA

Two DFA states are equivalent if they cannot be distinguished by any input string.

05. Regular Expressions

Every regular language has a regular expression. The state elimination method converts a DFA to a regex by progressively removing states and annotating transitions with regex labels.

06. Dead State (∅)

When ε-closure(move(S, a)) = ∅, the DFA transitions to a dead state ∅. This is a non-accepting trap state — once entered, the machine can never reach an accept state.
